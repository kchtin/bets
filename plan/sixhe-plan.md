# sixhe — Rust + JS Windows 桌面应用计划

## 1. 项目目标

- 用 Rust + JS 技术栈实现一个轻量 Windows 桌面应用。
- 核心流程：粘贴/输入原始下注文案 → 解析为标准注单 → 按设置拆单 → 自动复制结果。
- 打包为单个/小型 Windows 可执行文件，体积远小于 Electron / Node 方案。

## 2. 技术栈选择

| 层级 | 选型 | 理由 |
|------|------|------|
| 桌面框架 | Tauri 2.x | Rust 后端 + Web 前端，Windows 安装包可控制在 5MB 以内 |
| 构建工具 | Vite | 与现有 user-web 一致，开发体验好 |
| 前端 | 原生 HTML + CSS + JS | 界面简单，最小化依赖与打包体积；聚焦 Rust/JS 交互学习 |
| Rust 工具链 | cargo + tauri-cli | 标准 Rust 工具链 |
| 剪贴板 | Tauri Clipboard API | 跨平台原生能力，无需额外权限 |

## 4. 界面布局

整体采用左右两栏：左侧操作区，右侧设置区。

```
+------------------------------------------------+
|  左上区          |  右侧设置区                 |
|  [粘贴] [输入框] |  组数: [  ]                 |
+------------------+  最低金额: [  ]             |
|  左中区          |  [5x] [10x]                 |
|  [解析结果文本框]|  [拆单]                     |
|                  |                             |
+------------------+-----------------------------+
|  左下区          |                             |
|  [拆单结果文本框]|                             |
|  [拆单并复制]    |                             |
+------------------+-----------------------------+
```

### 4.1 左上区

- **粘贴按钮**：调用 Tauri Clipboard API，读取系统剪贴板文本并填充到输入框。
- **输入框**：可直接输入或编辑内容，作为解析源。

### 4.2 左中区

- **解析结果框**：实时/点击解析后，显示格式化的注单列表。
- 每行按 5 列对齐排版，使用等宽字体，例如：
  ```
  01*200   13*200   25*200   37*200   49*200
  02*200   14*200   26*200   38*200
  ```

### 4.3 左下区

- **拆单结果框**：显示拆单后的最终注单文本。每组内按号码从小到大排序。
- **拆单按钮**：执行拆单，并将结果自动写入剪贴板。

### 4.4 右侧设置区

- **组数输入框**：目标分组数量，正整数。
- **最低金额输入框**：每组最低金额，正整数。
- **5x / 10x 单选按钮**：控制拆单时金额的步长，必须选中其中一个。
  - `5x`：每份金额 = `最低金额 + 5n`
  - `10x`：每份金额 = `最低金额 + 10n`
  - 默认选中 `10x`。
- **状态日志**：右侧显示拼接的日志列表，自动滚动到最新，也可手动滚动查看历史。
- **拆单按钮**：与左下拆单按钮功能一致，触发拆单并复制。

## 5. 解析规则

输入为一段或多段下注文本，解析为 `号码*金额` 的标准格式。

### 5.1 支持的分隔符

号码之间：`，` `、` `,` `/` `.`

### 5.2 支持的金额连接符

号码与金额之间：`*` `个` `各`

### 5.3 号码类型

- 数字号码：`1` ~ `49`（或不限，按实际业务）
- 生肖号码：`鼠 牛 虎 兔 龙 蛇 马 羊 猴 鸡 狗 猪`

### 5.4 典型输入示例

| 输入 | 输出 |
|------|------|
| `10,20,30*10` | `10*10,20*10,30*10` |
| `10、20、30个10` | `10*10,20*10,30*10` |
| `10/20/30各10` | `10*10,20*10,30*10` |
| `鼠牛虎兔*5` | `鼠*5,牛*5,虎*5,兔*5` |
| `10，20*10` | `10*10,20*10` |

### 5.5 解析策略

1. 先提取金额：匹配末尾的 `连接符 + 数字`，如 `*10`、`个10`、`各10`。
2. 剩余部分按分隔符拆分为号码数组。
3. 每个号码后追加 `*金额`。
4. 多个片段可换行或空格分隔，逐段解析后合并。
5. 生肖号码通过 `ZODIAC_MAP_2026` 展开为对应数字号码；如规则变化，替换该映射表即可。

### 5.6 错误处理

- 无法解析时左中区显示提示：`解析失败：未识别金额或号码`。
- 号码为空或金额为 0/负数时给出明确提示。

## 6. 拆单规则

### 6.1 输入

- 左中区解析后的标准注单列表，例如：
  `[{code: '10', amount: 10}, {code: '11', amount: 10}, {code: '12', amount: 10}, {code: '01', amount: 10}]`

### 6.2 设置项

- **组数**：拆成多少组，例如 5。
- **最低金额**：每组的最小金额，例如 10。
- **倍数**：5x / 10x 是单选步长，控制拆单时每份金额的增量（5 或 10），必须选中其一。

### 6.3 拆单目标

将每个号码的原始金额，拆分为若干子注单，使得：

1. 总组数不超过设置的组数。
2. 每组总金额 ≥ 最低金额。
3. 同一号码的金额在拆分后总量保持不变。
4. 输出格式与解析结果一致：`号码*金额`，多组用换行或指定分隔符区分。

### 6.4 拆单算法草案

采用“按号码随机拆单”策略：

1. `5x` / `10x` 控制拆单时的金额步长（step）：
   - `5x`：每份金额 = `最低金额 + 5n`
   - `10x`：每份金额 = `最低金额 + 10n`
2. 对每个号码独立处理：
   - 随机决定该号码分配到几组。
   - 随机选择具体哪些组。
   - 将该号码的原始金额随机拆分为若干份，每份金额满足：
     - 不低于设置的最低金额；
     - 等于 `最低金额 + n * step`；
     - 所有份之和等于该号码的原始金额。
3. 一个号码不必出现在所有组中。
4. 最终输出按组分行，组内用 `,` 分隔。

> 注：若号码原始金额不是 step 的整数倍，则无法拆分，会提示错误。

示例：

- 输入：`10*10,11*10,12*10,1*10`（总金额 40）
- 组数：2，最低金额：10
- 结果：
  ```
  10*10,11*10
  12*10,1*10
  ```

- 输入：`10*10,11*10,12*40,1*10`
- 组数：2，最低金额：25
- target = max(25, ceil(70/2)=35) = 35，`12*40` 会被拆分为两份。
- 输出示例：
  ```
  11*10,10*10,12*21
  12*19,1*10
  ```
  （允许拆开一个号码的金额到两组）

### 6.5 复制行为

拆单完成后，自动将结果文本写入系统剪贴板，并给出轻量提示（如按钮文字短暂变“已复制”）。

## 7. 数据流

```
用户粘贴/输入
    │
    ▼
左上输入框 ──► JS parser ──► 左中区标准格式
                                │
                5x / 10x 步长 ──┤
                                ▼
                        JS splitter
                                │
                                ▼
                        左下区结果 + 写入剪贴板
```

## 8. 开发步骤

### Step 1：初始化 Vite 项目 ✅

1. 项目根目录即 `sixhe/`。
2. 使用 `npm create vite@latest sixhe -- --template vanilla` 初始化前端项目。
3. 安装 `vitest` 作为测试框架。

### Step 2：实现前端骨架 ✅

1. 编写 `index.html` 左右布局。
2. 编写 `style.css` 基础样式。
3. 在 `main.js` 中绑定按钮事件，使用浏览器 Clipboard API 临时实现粘贴/复制。

### Step 3：实现解析模块 ✅

1. 在 `parser.js` 中实现 `parseBetText(text)`。
2. 覆盖分隔符、连接符、数字/生肖号码。
3. 添加单元测试到 `test/parser.test.js`。

### Step 4：实现拆单模块 ✅

1. 在 `splitter.js` 中实现 `splitBets(bets, groups, minAmount)`。
2. 支持 5x/10x 单选步长。
3. 添加单元测试到 `test/splitter.test.js`。

### Step 5：接入 Tauri 剪贴板 ⏳

1. 在 Rust 侧暴露或直接使用 `@tauri-apps/api/clipboard`。
2. 实现粘贴读取与结果写入。

> 当前环境未安装 Rust，Tauri 相关步骤需后续补齐。

### Step 6：Windows 打包 ⏳

1. 配置 `tauri.conf.json` 的 Windows 构建选项。
2. 运行 `cargo tauri build --target x86_64-pc-windows-msvc`。
3. 输出 `.msi` 与/或 `.exe` 安装包。

> 当前环境未安装 Rust，Windows 打包需在安装 Rust 工具链后于 Windows 或交叉编译环境执行。

## 10. 风险与注意事项

1. **Windows 打包环境**：Tauri 在 Windows 上需要 WebView2 运行时，目标 Win10/11 通常已内置；旧系统需安装。
2. **生肖映射**：已在 `src/parser.js` 中抽象为 `ZODIAC_MAP_2026`，采用 2026 丙午年马年起头方案；若业务规则变化，直接替换该对象即可。
3. **Rust 工具链缺失**：当前 macOS 环境未检测到 `cargo`/`rustc`，Tauri 后端初始化、`src-tauri/` 构建与 Windows 打包均无法执行，需后续安装 Rust 工具链后补齐。
4. **UI 体积**：原生 HTML/CSS/JS 可保持最小；如后续需要组件库再引入。
