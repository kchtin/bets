# sixhe — 注单解析与拆单工具计划

## 1. 项目目标

- 实现一个轻量桌面应用，核心流程：粘贴/输入原始下注文案 → 解析为标准注单 → 按设置拆单 → 自动复制结果。
- 提供两套技术栈实现：原生 JS 版（tauri-js）与 React + TypeScript 版（react-ts）。
- 打包为小型 Windows 可执行文件。

## 2. 技术栈选择

| 层级 | 原生 JS 版 | React + TypeScript 版 |
|------|-----------|----------------------|
| 桌面框架 | Tauri 2.x | Tauri 2.x |
| 前端 | 原生 HTML + CSS + JS | React 19 + TypeScript |
| UI 方案 | 手写 CSS | Tailwind CSS + shadcn/ui 风格组件 |
| 构建工具 | Vite | Vite |
| 测试 | Vitest | Vitest |
| 剪贴板 | 浏览器 Clipboard API | 浏览器 Clipboard API |

## 5. 解析规则

输入为一段或多段下注文本，解析为 `号码*金额` 的标准格式。

### 5.1 支持的分隔符

号码之间：`,`, `，`, `、`, `/`, `.`

### 5.2 支持的金额连接符

号码与金额之间：`*`, `个`, `各`

### 5.3 号码类型

- 数字号码：`1` ~ `49`
- 生肖号码：`鼠 牛 虎 兔 龙 蛇 马 羊 猴 鸡 狗 猪`

### 5.4 典型输入示例

| 输入 | 输出 |
|------|------|
| `10,20,30*10` | `10*10,20*10,30*10` |
| `10、20、30个10` | `10*10,20*10,30*10` |
| `鼠牛虎兔*5` | `鼠→对应号码*5` |
| `01.13.25.37.49*200` | `01*200,13*200,...` |

### 5.5 解析策略

1. 先提取金额：匹配末尾的 `连接符 + 数字`。
2. 剩余部分按分隔符拆分为号码数组。
3. 生肖号码通过 `ZODIAC_MAP_2026` 展开为对应数字号码。
4. 每个号码后追加 `*金额`。

### 5.6 格式化输出

解析结果按 5 列对齐排版，使用等宽字体，例如：

```
01*200   13*200   25*200   37*200   49*200
02*200   14*200   26*200   38*200
```

## 6. 拆单规则

### 6.1 输入

解析后的标准注单列表，例如：
`[{code: '08', amount: 200}, {code: '01', amount: 200}]`

### 6.2 设置项

- **组数**：拆成多少组，默认 5。
- **最低金额**：每组中每个号码的最小金额。
- **倍数步长**：`5x` 或 `10x`，单选必中其一，默认 `10x`。

### 6.3 拆单目标

- 每个号码的金额被随机拆分到若干组。
- 每组中该号码的金额 ≥ 最低金额。
- 每组中该号码的金额 = `最低金额 + n * 步长`。
- 所有组中该号码的金额之和 = 原始金额。
- 每组内按号码从小到大排序。

### 6.4 示例

输入：`08.01*200`，组数 5，最低金额 10，步长 10

可能输出：

```
01*60,08*90
01*50,08*30
01*90,08*40
01*40,08*70
08*50,01*60
```

## 7. 自动拆单与复制

- 左上输入框内容变化 → 自动解析并拆单
- 组数/最低金额变化 → 自动拆单
- `5x`/`10x` 切换 → 自动拆单
- 点击解析结果框 → 复制解析结果
- 点击拆单结果框 → 复制拆单结果
- 右侧日志区拼接显示，自动滚动到最新

## 8. 数据流

```
用户粘贴/输入
    │
    ▼
输入框 ──► parser ──► 格式化解析结果
                          │
        组数/最低金额/步长变化 ──┤
                          ▼
                      splitter
                          │
                          ▼
                  拆单结果 + 日志
```

## 10. 风险与注意事项

1. **Artifact 存储额度**：GitHub Actions Artifact 免费额度 500MB，建议日常构建保留 1 天，正式发布使用 Release。
2. **热更新/热重载**：当前为本地打包方案；若需免打包更新，可将 `frontendDist` 改为 GitHub Pages / Gitee Pages 等远程地址。
3. **生肖映射规则**：采用 2026 丙午年马年起头方案；若业务规则变化，替换 `ZODIAC_MAP_2026` 即可。
4. **Rust 工具链**：本地 macOS 未安装 Rust，Windows 打包依赖 CI；本地构建需安装 Rust + MSVC。
